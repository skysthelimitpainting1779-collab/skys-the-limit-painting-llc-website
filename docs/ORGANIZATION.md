# Repository Organization

## Branches

- `main` is the production branch.
- Feature work should use short-lived branches such as `codex/add-lead-docs` or `feature/gallery-refresh`.
- Release tags should use `vMAJOR.MINOR.PATCH`.

## Labels

Recommended issue and pull request labels:

- `bug`
- `enhancement`
- `documentation`
- `dependencies`
- `maintenance`
- `security`
- `seo`
- `performance`
- `content`

## Review Expectations

- Pull requests should explain what changed and how it was validated.
- UI changes should include screenshots or a deployment preview.
- Production-sensitive changes should mention rollback notes.
